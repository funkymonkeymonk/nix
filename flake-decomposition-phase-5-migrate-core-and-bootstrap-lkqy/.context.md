# Problem
core (Darwin) and bootstrap (NixOS) are minimal configs used for initial system setup. They use only modules/common/core.nix and modules/common/options.nix. These are the simplest migrations — minimal modules, no user config, no services.

Decision: Use raw nixpkgs.lib.nixosSystem / nix-darwin.lib.darwinSystem for these two. Adding a "minimal" flag to the library helpers adds complexity for just two low-value targets. bootstrap and core are intentionally minimal and don't benefit from the library's home-manager/opnix/modules inclusion.
