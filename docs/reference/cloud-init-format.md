# Cloud-init Format

Reference for the cloud-init YAML files used by the MicroVM host system.

## Host-level Cloud-init

**Location:** `/etc/cloud-init.yaml`

Defines system-wide configuration and MicroVM definitions.

### MicroVM Definitions

```yaml
microvms:
  - name: matrix
    flake: .#microvm.nixosConfigurations.matrix
    ipAddress: 192.168.83.15
    autoStart: true
  - name: openclaw
    flake: .#microvm.nixosConfigurations.openclaw
    ipAddress: 192.168.83.16
    autoStart: true
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | yes | MicroVM name. Must match a `microvm.nixosConfigurations` entry in the flake. |
| `flake` | string | yes | Flake attribute path. Format: `.#microvm.nixosConfigurations.<name>` |
| `ipAddress` | string | yes | Static IP on the bridge network (`192.168.83.0/24`). |
| `autoStart` | bool | no | Start on boot. Default: `true`. |

### Other Fields

Standard cloud-init fields are also supported:

| Field | Type | Description |
|-------|------|-------------|
| `hostname` | string | Host system hostname |
| `timezone` | string | System timezone (e.g. `America/New_York`) |
| `locale` | string | System locale (e.g. `en_US.UTF-8`) |
| `ssh_authorized_keys` | list | SSH keys for root access |
| `nix.target` | string | Nix flake target name |
| `nix.branch` | string | Git branch for flake |
| `nix.impure` | bool | Enable impure evaluation |
| `nix.roles` | list | Enabled Nix roles |

## Per-VM Cloud-init

**Location:** `/etc/cloud-init/<hostname>.yaml` (generated at build time)

Mounted into each VM via virtiofs at `/etc/cloud-init/`.

### Format

```yaml
#cloud-config
hostname: <name>
```

Currently contains only the hostname. Future: secrets, onecli configuration.

### Location on Host

Generated by the `microvm-host` role module via `environment.etc`. Files are placed at `/etc/cloud-init/<hostname>.yaml` on the host and mounted into the corresponding VM.

## CLI Commands

Manage cloud-init via `nix-cloud-init`:

```bash
# Host-level
nix-cloud-init show
sudo nix-cloud-init set hostname myserver
sudo nix-cloud-init set nix.roles microvm-host

# MicroVM management
nix-cloud-init microvm list
sudo nix-cloud-init microvm add <name> <flake> <ip> [autoStart]
sudo nix-cloud-init microvm remove <name>
sudo nix-cloud-init microvm generate
```

> For usage examples, see [Manage MicroVMs](../how-to/manage-microvms.md).
