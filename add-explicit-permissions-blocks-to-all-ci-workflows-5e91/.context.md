# Problem
Four workflow files lack explicit `permissions:` blocks, which means they run with default (broad) token permissions. The other workflows already have explicit permissions set.

Missing from:
- `.github/workflows/build-iso.yml`
- `.github/workflows/cache-maintenance.yml`
- `.github/workflows/main-build.yml`
- `.github/workflows/pr-validation.yml`

Present in: `flake-update.yml`, `flakehub-publish-rolling.yml`, `opencode.yml`, `release.yml`, `weekly-release.yml`

# Acceptance Criteria
- [ ] Each of the four workflows has a top-level `permissions:` block with least-privilege settings
- [ ] `build-iso.yml`: needs `contents: read` (reads repo, writes nothing)
- [ ] `cache-maintenance.yml`: needs `contents: read` (scheduled cache clear)
- [ ] `main-build.yml`: needs `contents: read` (builds and pushes to Cachix, not GitHub)
- [ ] `pr-validation.yml`: needs `contents: read`, `pull-requests: read` (reads PR context)
- [ ] All existing CI workflows continue to pass after the change
