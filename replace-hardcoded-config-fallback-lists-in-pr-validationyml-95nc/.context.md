# Problem
When no `.drv-cache` exists (first run, or after cache eviction), `pr-validation.yml` falls back to hardcoded lists instead of dynamically discovering configs:

Line ~202: `DARWIN_CHANGED='["core","darwin-server","wweaver"]'`
Line ~228: `NIXOS_CHANGED='["bootstrap","installer-iso","type-desktop","type-server","zero"]'`

This means adding a new machine target to flake.nix requires also updating these fallback lists manually. `main-build.yml` already does dynamic discovery via `nix eval .#darwinConfigurations` / `.#nixosConfigurations` — the same approach should be used here.

# Acceptance Criteria
- [ ] When no cache exists, `pr-validation.yml` dynamically discovers Darwin configs via `nix eval .#darwinConfigurations --apply 'builtins.attrNames' --json`
- [ ] When no cache exists, `pr-validation.yml` dynamically discovers NixOS configs the same way
- [ ] Hardcoded fallback lists (`["core","darwin-server","wweaver"]` and `["bootstrap","installer-iso","type-desktop","type-server","zero"]`) are removed
- [ ] Adding a new target to `flake.nix` is automatically picked up without touching CI files
- [ ] CI still passes
