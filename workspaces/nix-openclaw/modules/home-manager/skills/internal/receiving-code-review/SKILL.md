# Receiving Code Review

## Overview

Process code review feedback with technical rigor.

## When to Use

- You've received PR feedback
- Need to address review comments
- Evaluating suggested changes

## Process

1. **Read all comments** before responding
2. **Understand** the intent behind each comment
3. **Ask for clarification** if needed
4. **Implement changes** or explain why not
5. **Verify** the changes work as expected

## Key Principles

- Don't take feedback personally
- Ask questions to understand
- Test your changes thoroughly
- Keep an open mind
