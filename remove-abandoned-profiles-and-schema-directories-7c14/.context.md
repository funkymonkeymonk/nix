# Goal
Delete the unused `profiles/` and `schema/` directories that were part of an unfinished profile system.

## Prerequisites
- None

# Acceptance Criteria
- [ ] Verify nothing in `flake.nix` imports from `profiles/` or `schema/`
- [ ] Verify no GitHub Actions, scripts, or docs reference `profiles/` or `schema/`
- [ ] Delete `profiles/monkey/` directory
- [ ] Delete `profiles/wweaver/` directory
- [ ] Delete `schema/` directory
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Files
- `profiles/monkey/flake.nix`
- `profiles/monkey/profile.nix`
- `profiles/wweaver/flake.nix`
- `profiles/wweaver/profile.nix`
- `schema/flake.nix`
- `schema/flake-module.nix`
- `schema/modules/profile.nix`
- `schema/modules/validator.nix`

# Notes
The `schema/` validator exports `mkProfile` which is used by `profiles/monkey/flake.nix` and `profiles/wweaver/flake.nix`, but the main repo flake never consumes these profiles. They appear to be an experimental user-config system that was never integrated. The `mkUser` helper in `flake.nix` already serves a similar purpose.
