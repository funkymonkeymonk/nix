# Goal
Complete the flake decomposition migration by removing parallel v1/v2 configurations and promoting v2 as the canonical implementation.

## Prerequisites
- "Delete _mkMicrovmV2 dead code in flake.nix" must be done

# Acceptance Criteria
- [ ] Remove v1 MicroVM configs (`microvm.nixosConfigurations.{dev-vm,openclaw,matrix,media-center}`) and rename v2 configs to drop `-v2` suffix
- [ ] Remove v1 NixOS configs (`zero`, `type-server`, `type-server-arm`, `type-desktop`) and promote v2 configs
- [ ] Remove v1 Darwin configs (`darwin-server`, `core`) and promote v2 configs (`darwin-server-v2`, `core-v2`)
- [ ] Update `packages` attrset in flake.nix to reference new canonical MicroVM names
- [ ] Update all documentation references from `-v2` suffix to canonical names
- [ ] Update `.github/workflows/` if any reference v2-specific flake outputs
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`
- [ ] Verify at least one v2 config evaluates: `nix build .#nixosConfigurations.zero.config.system.build.toplevel --dry-run`

# Files

# Notes
The v2 library (`library/lib/mk-system.nix`) uses `mkDarwinSystem` and `mkNixosSystem` which centralize nixpkgs config, overlays, and module imports. The v1 configs inline all of this. Migration was intentionally parallel for safety, but the v2 configs have been stable long enough to promote.
