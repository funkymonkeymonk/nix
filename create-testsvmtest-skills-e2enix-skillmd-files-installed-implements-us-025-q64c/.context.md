# Goal
Create tests/vm/test-skills-e2e.nix that boots a NixOS VM with opencode role enabled
and verifies SKILL.md files are installed to ~/.config/opencode/skills/. Implements US-025.

# Implementation Notes

## Use mkVmTest helper
Requires full ./modules tree for home-manager skills install module. See parent yak for mkVmTest design.

## How skills are installed
Skills are installed via home-manager's home.file entries generated by modules/home-manager/skills/install.nix.
The home-manager activation writes files to ~/.config/opencode/skills/<skill-name>/SKILL.md.
The superpowersPath input must be available — use a stub or the actual superpowers flake input.

## Handling superpowersPath in tests
myConfig.skills.superpowersPath requires a flake input (the superpowers flake).
Options:
1. Pass `self` to the test (already done for vm tests via `{inherit pkgs self;}`) and use inputs.superpowers
2. Create a minimal stub path with a fake SKILL.md for test purposes
3. Set enabledRoles = [] to skip external skills and only test internal skills

Recommended: pass the actual superpowers flake input via specialArgs, same pattern as type-server uses.

## Test cases

### skills-opencode-installed
- Enable myConfig.roles.opencode.enable = true
- Set myConfig.skills.enabledRoles = ["opencode"]
- Boot and assert: /home/testuser/.config/opencode/skills/ directory exists
- Assert: at least one SKILL.md file present (e.g., ls ~/.config/opencode/skills/*/SKILL.md succeeds)

### skills-role-filtering
- Enable foundation + developer roles
- Assert: developer-specific skills (e.g., jj, debugging) are present
- Assert: opencode-only skills are NOT present

## Note on US-025
US-025 in prd.json asks for devenv task agent-skills:validate to check SKILL.md files.
This VM test is the integration test backing that. The devenv task can be a thin wrapper
that calls `nix build .#checks.x86_64-linux.skills-e2e-opencode`.
