# Problem
`modules/common/options.nix` is 1,288 lines covering all options: users, roles, services (ollama, vane, openclaw), apps (mods, claude, pi, sketchybar), system settings, etc. This makes it hard to find options for a specific feature and the file grows with every new module.

# Acceptance Criteria
- [ ] Options are split into per-feature files, e.g.:
  - `modules/common/options/roles.nix` — role enable options
  - `modules/common/options/services.nix` — ollama, vane, openclaw, etc.
  - `modules/common/options/apps.nix` — mods, claude, pi, sketchybar, etc.
  - `modules/common/options/system.nix` — core system settings
- [ ] `modules/common/options.nix` either becomes a thin import aggregator or is removed
- [ ] All existing machine configs still evaluate: `devenv tasks run check:all` passes
- [ ] No option paths change (no breaking changes to `myConfig.*`)

# Files
- `modules/common/options.nix` (split or convert to aggregator)
- `modules/common/options/` (new directory with per-feature files)
