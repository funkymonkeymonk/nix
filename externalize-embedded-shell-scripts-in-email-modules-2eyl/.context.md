# Notes
`jj-autosync.nix` already uses this pattern:
```nix
sharedLibContent = builtins.readFile ./jj-autosync-lib.sh;
jjAutosyncContent = withSharedLib (builtins.readFile ./jj-autosync.sh);
```
The email modules embed ~500 lines of bash directly in Nix strings, which:
- Breaks shell syntax highlighting in editors
- Makes the Nix files unreadably long
- Prevents shellcheck from running on the scripts
Externalizing fixes all three issues.
