# Goal
Add tests for `library/lib/mk-system.nix` to prevent regressions in `mkDarwinSystem` and `mkNixosSystem` helpers.

## Prerequisites
- None

# Acceptance Criteria
- [ ] Create `tests/test-library.nix` with tests for:
  - `mkDarwinSystem` produces correct `networking.hostName`
  - `mkDarwinSystem` includes expected overlays (stable, devenv, zellij-pane-tracker)
  - `mkDarwinSystem` sets `nixpkgs.config.allowUnfree = true`
  - `mkNixosSystem` produces correct `networking.hostName`
  - `mkNixosSystem` includes `home-manager.nixosModules.home-manager`
  - `mkNixosSystem` includes `opnix.homeManagerModules.default` in sharedModules
  - `mkNixosSystem` sets `nixpkgs.hostPlatform` correctly
  - Overrides are applied at correct priority (mkOverride 50)
- [ ] Wire `test-library.nix` into `tests/default.nix`
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Files
- `tests/test-library.nix` (new)
- `tests/default.nix`

# Notes
These helpers centralize nixpkgs config, overlays, and module imports. A regression here affects every v2 system. Use `lib.evalModules` with stubbed inputs to test without building actual systems. Follow the pattern from `tests/test-vllm-mlx.nix` for stubbing inputs.
