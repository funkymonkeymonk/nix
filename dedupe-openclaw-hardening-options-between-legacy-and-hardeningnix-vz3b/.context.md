# Goal
Eliminate the duplicated hardening option schema between `legacy.nix` and `hardening.nix`.

## Prerequisites
- "Remove dead OpenClaw legacy modules" must be done

# Acceptance Criteria
- [ ] If legacy module is removed first: this yak becomes a no-op (hardening.nix is the only copy left)
- [ ] If this yak is done first: extract hardening options into `modules/services/openclaw/hardening-options.nix`
- [ ] Import the shared options from both `legacy.nix` and `hardening.nix`
- [ ] Ensure no change in option defaults or semantics
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Files
- `modules/services/openclaw/legacy.nix`
- `modules/services/openclaw/hardening.nix`
- `modules/services/openclaw/hardening-options.nix` (new, if needed)

# Notes
Both files define identical options: `noNewPrivileges`, `protectSystem`, `protectHome`, `privateTmp`, `protectKernelTunables`, `protectKernelModules`, `protectControlGroups`, `restrictSUIDSGID`, `privateDevices`, `restrictNamespaces`, `restrictAddressFamilies`, `systemCallFilter`, `memoryDenyWriteExecute`, `lockPersonality`. The defaults and descriptions are the same. Best path: remove legacy first, then this yak is trivial.
