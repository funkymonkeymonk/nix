# Problem
`modules/home-manager/jj-autosync.nix` (or similar) contains embedded bash scripts as Nix strings. This makes the bash hard to read, test, and lint in isolation. The skill iterating-nix-embedded-scripts exists for exactly this pattern.

# Acceptance Criteria
- [ ] Bash scripts currently embedded in jj-autosync.nix are extracted to `.sh` files alongside the module (e.g., `modules/home-manager/jj-autosync-daemon.sh`)
- [ ] The Nix module reads the scripts via `builtins.readFile ./jj-autosync-daemon.sh` (or uses `writeShellApplication`)
- [ ] The scripts are syntax-checked by shellcheck in CI (add to `check:lint` or a new task)
- [ ] Functionality is unchanged — `devenv tasks run test:nixos-eval` and `test:darwin-eval` pass

# Files
- `modules/home-manager/jj-autosync.nix` (replace embedded strings with readFile)
- `modules/home-manager/jj-autosync-daemon.sh` (new file, extracted script)
