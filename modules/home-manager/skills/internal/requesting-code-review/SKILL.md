# Requesting Code Review

## Overview

Properly request code reviews and prepare PRs.

## When to Use

- Code is ready for review
- Creating a new PR
- Following up on pending reviews

## Process

1. **Self-review first** - check your own code
2. **Write clear description** - what and why
3. **Highlight key changes** - what reviewers should focus on
4. **Request specific reviewers** if appropriate
5. **Respond promptly** to feedback

## Key Principles

- Keep PRs focused and reasonably sized
- Provide context in descriptions
- Link to related issues/tickets
- Be responsive to feedback
