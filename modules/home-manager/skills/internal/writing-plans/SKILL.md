# Writing Plans

## Overview

Create detailed implementation plans from specs and requirements.

## When to Use

- Starting implementation of approved feature
- Breaking down large tasks
- Handing off work to others

## Structure

1. **Context**: What are we building and why?
2. **Requirements**: Functional and non-functional
3. **Approach**: High-level strategy
4. **Tasks**: Specific, verifiable steps
5. **Testing**: How to verify completion
6. **Risks**: What could go wrong?

## Task Format

Each task should be:
- Specific and actionable
- Include verification criteria
- Estimated time
- Dependencies noted
