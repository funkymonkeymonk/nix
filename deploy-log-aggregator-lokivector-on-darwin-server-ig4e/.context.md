# Goal
Set up centralized log collection on protoman (darwin-server) to aggregate logs from MicroVMs and host services.

# Components
- Vector: Log shipper (runs on host and in MicroVMs)
- Loki: Log storage and query (runs in Colima/Docker)

# Acceptance Criteria
- [ ] Vector agent running on darwin-server collecting system logs
- [ ] Vector agent in MicroVMs forwarding logs to host
- [ ] Loki instance running in Colima VM
- [ ] Logs queryable via Grafana or CLI
- [ ] Log retention policy configured (7 days default)

# Files to modify
- modules/services/vector/darwin.nix (new)
- modules/services/loki/darwin.nix (new)
- targets/darwin-server/default.nix (enable services)

# Notes
MicroVMs already have network access to host at 192.168.83.1.
Use existing Colima infrastructure for containerized Loki.
