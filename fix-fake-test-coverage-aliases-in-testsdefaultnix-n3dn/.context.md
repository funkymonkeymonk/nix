# Goal
Remove the fake coverage aliases in `tests/default.nix` where multiple check names point to the same derivation, misleading CI and coverage metrics.

## Prerequisites
- None

# Acceptance Criteria
- [ ] Audit `tests/default.nix` for all aliases that map multiple names to the same test derivation
- [ ] Remove redundant aliases, keeping only one canonical name per test
- [ ] Update `flake.nix` `checks` attrset to reference only the canonical names
- [ ] Ensure `nix flake check` still passes after deduplication
- [ ] Add real tests for the missing coverage areas (role-composition, role-cascades, etc.) OR remove the aliases and document the gap
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Files
- `tests/default.nix`
- `flake.nix` (checks attrset)
- `tests/test-roles.nix` (may need expansion)

# Notes

```nix
all-role-tests = testRoles.allRoleTests;

role-composition = testRoles.allRoleTests;
role-packages = testRoles.allRoleTests;
```
