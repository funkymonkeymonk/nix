# Goal
Standardize the service registry pattern used across Darwin service modules by extracting a shared helper.

## Prerequisites
- "Extract Darwin user/homeDir helper for service modules" should be done first

# Acceptance Criteria
- [ ] Add `registerService` helper to `modules/common/darwin-lib.nix` (or new `modules/common/service-registry.nix`):
  ```nix
  registerService = name: { port, launchdLabel, errorLog }: {
    myConfig.serviceRegistry.${name} = { inherit name port launchdLabel errorLog; };
  };
  ```
- [ ] Refactor `modules/services/bifrost/darwin.nix` to use `registerService`
- [ ] Refactor `modules/services/vane/darwin.nix` to use `registerService`
- [ ] Refactor `modules/services/caddy/darwin.nix` to use `registerService`
- [ ] Refactor `modules/services/searxng/darwin.nix` to use `registerService`
- [ ] Refactor `modules/services/vllm-mlx/darwin.nix` to use `registerService`
- [ ] Ensure `dnsmasq` registration in caddy also uses the helper
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Notes
Every Darwin service manually registers itself:
```nix
myConfig.serviceRegistry = optionalAttrs cfg.enable {
  bifrost = {
    name = "Bifrost";
    port = cfg.port;
    launchdLabel = "com.bifrost.service";
    errorLog = "/tmp/bifrost.error.log";
  };
};
```
A helper would reduce this to one line and enforce consistent naming conventions.
