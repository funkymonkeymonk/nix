# Goal
Eliminate the duplicated `primaryUser` / `darwinHomeDir` pattern across all Darwin service modules by extracting a shared helper.

## Prerequisites
- None

# Acceptance Criteria
- [ ] Create `modules/common/darwin-lib.nix` with a `darwinUserEnv` helper:
  ```nix
  darwinUserEnv = config: let
    user = if config.myConfig.users != []
      then (builtins.head config.myConfig.users).name
      else "root";
  in { name = user; home = "/Users/${user}"; };
  ```
- [ ] Refactor `modules/services/bifrost/darwin.nix` to use the helper
- [ ] Refactor `modules/services/vane/darwin.nix` to use the helper
- [ ] Refactor `modules/services/caddy/darwin.nix` to use the helper
- [ ] Refactor `modules/services/searxng/darwin.nix` to use the helper
- [ ] Refactor `modules/services/vllm-mlx/darwin.nix` to use the helper
- [ ] Refactor `modules/services/lume/darwin.nix` to use the helper
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Notes
Each Darwin service currently copy-pastes:
```nix
primaryUser = if config.myConfig.users != [] then (builtins.head config.myConfig.users).name else "monkey";
darwinHomeDir = "/Users/${primaryUser}";
```
Some default to "monkey", some to "root". Standardize on "root" as the fallback in the helper for consistency.
