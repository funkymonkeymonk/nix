# Goal
Use npx skills (https://www.npmjs.com/package/skills) to handle external/community
skill installation, filling the gap left by the source.type = "external" placeholder
in manifest.nix.

# Why
- Internal skills are well-served by the Nix manifest approach (declarative, role-gated, pinned)
- External skills from GitHub repos have no working installation path today
- npx skills already knows the correct install paths for 44+ agents
- It handles symlink vs copy, project vs global scope, updates, and discovery

# Approach
- Add a Nix activation script that runs `npx skills add` for manifest entries with source.type = "external"
- Map the manifest's url field to npx skills source format (GitHub shorthand or URL)
- Use --global --yes --agent flags to target the correct agents non-interactively
- Run on system activation (darwin-rebuild switch / nixos-rebuild switch)
- Consider caching to avoid re-downloading on every rebuild

# Acceptance Criteria
- [ ] External skills in manifest.nix actually install (not just placeholder text)
- [ ] Works for OpenCode, Claude Code, and Pi agents
- [ ] Idempotent -- re-running doesn't duplicate or break anything
- [ ] npx/node available via Nix (nodejs in buildInputs or systemPackages)
