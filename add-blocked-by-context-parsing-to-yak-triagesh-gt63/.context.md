# Goal

Enhance `yak-triage.sh` to parse the `## Blocked By` section in yak contexts,
filtering out yaks that are semantically blocked even if their yx metadata
(state, tags, children) says they're actionable.

# Background

Currently `yak-triage.sh` checks:
- State is "todo"
- Yak is a leaf (no pending children)
- No blocking tags (@needs-human, @blocked, etc.)

But it doesn't parse the yak's context markdown for `## Blocked By` sections,
which leads to false positives — reporting yaks as "actionable" when they're
actually waiting on another yak that isn't tracked in the yx DAG.

Example: "Scaffold Go module" has `## Blocked By: Author initial ADRs...` in
its context, but triage reports it as actionable because there's no @blocked tag.

# Acceptance Criteria

- [ ] `yak-triage.sh` fetches context for each candidate yak
- [ ] If context contains `## Blocked By` with non-empty content, yak is excluded
- [ ] Performance: Only fetch context for yaks that pass the initial metadata filter
- [ ] Add `--include-blocked` flag to show blocked yaks for debugging
- [ ] Update script header comment to document the new behavior
