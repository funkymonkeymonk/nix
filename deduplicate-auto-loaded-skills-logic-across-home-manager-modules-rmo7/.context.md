# Goal
Extract the duplicated manifest-import / role-filter / auto-load logic from `claude-code.nix` and `pi-coding-agent.nix` into a shared helper.

## Prerequisites
- None

# Acceptance Criteria
- [ ] Add `mkAutoLoadSkills` function to `modules/home-manager/lib.nix` that:
  - Imports `./skills/manifest.nix`
  - Reads `enabledRoles` from config
  - Filters skills by role membership
  - Filters to `autoLoad = true` skills
  - Resolves internal vs superpowers sources
  - Concatenates SKILL.md content with `---` separators
- [ ] Refactor `modules/home-manager/claude-code.nix` to use `hmLib.mkAutoLoadSkills`
- [ ] Refactor `modules/home-manager/pi-coding-agent.nix` to use `hmLib.mkAutoLoadSkills`
- [ ] Ensure the `hasAutoLoadSkills` boolean is also returned by the helper
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Files
- `modules/home-manager/lib.nix`
- `modules/home-manager/claude-code.nix`
- `modules/home-manager/pi-coding-agent.nix`

# Notes
Both modules contain ~30 lines of nearly identical logic:
```nix
manifest = import ./skills/manifest.nix;
enabledRoles = skillsCfg.enabledRoles or [];
enabledSkills = lib.filterAttrs (_name: skill: lib.any (role: lib.elem role skill.roles) enabledRoles) manifest;
autoLoadSkills = lib.filterAttrs (_name: skill: skill.autoLoad or false) enabledSkills;
autoLoadContent = lib.concatStringsSep "\n\n---\n\n" (lib.mapAttrsToList (name: skill: ...) autoLoadSkills);
```
This is a pure function of `config` and should live in the shared library.
