# Goal
Add option-level tests for modules that currently have no test coverage, preventing regressions in option defaults and structure.

## Prerequisites
- None

# Acceptance Criteria
- [ ] Create `tests/test-claude-code.nix` — test claude-code option defaults (enable, includeCoAuthoredBy, mcpServers, agents, commands, hooks)
- [ ] Create `tests/test-pi.nix` — test pi option defaults (enable, settings, agentsMd, models, prompts, skills, extensions, npmPackages, themes)
- [ ] Create `tests/test-bifrost.nix` — test bifrost option defaults (enable, port, host, logLevel, appDir, upstreams attrset)
- [ ] Create `tests/test-caddy.nix` — test caddy option defaults (enable, port, dataDir, hosts)
- [ ] Create `tests/test-searxng.nix` — test searxng option defaults (enable, port, secretKey)
- [ ] Create `tests/test-lume.nix` — test lume option defaults (enable, port, enableBackgroundService, prePullImages)
- [ ] Create `tests/test-email.nix` — test email-agent and email-backup option defaults
- [ ] Wire all new test files into `tests/default.nix`
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Notes
Follow the pattern from `tests/test-vllm-mlx.nix`:
1. Define stub modules for `nixpkgs.hostPlatform` and `_module.args`
2. Use `lib.evalModules` to evaluate options with defaults
3. Use `pkgs.runCommand` with shell conditionals to assert defaults
4. Export tests in an attrset and wire into `tests/default.nix`
