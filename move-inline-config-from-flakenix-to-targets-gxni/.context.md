# Problem
`flake.nix` contains large inline `myConfig` blocks for multiple machines (`wweaver`, `zero`, `MegamanX`, and others). The pattern of importing from `targets/<name>/` already exists but is inconsistently applied — some machines have a `targets/` directory but still have significant config inline in `flake.nix`.

This makes `flake.nix` very long and hard to navigate. Machine-specific config should live in `targets/<name>/`.

# Acceptance Criteria
- [ ] Each machine's `myConfig` block is moved to its `targets/<name>/default.nix` (or a `targets/<name>/config.nix`)
- [ ] `flake.nix` only contains the `nixosSystem`/`darwinSystem` call with module imports — no inline `myConfig`
- [ ] All machines still build: `devenv tasks run check:all` passes
- [ ] No regressions in any existing test

# Files
- `flake.nix` (remove inline myConfig blocks)
- `targets/wweaver/default.nix` (receive wweaver config)
- `targets/zero/default.nix` (receive zero config)
- `targets/MegamanX/default.nix` (receive MegamanX config, create if needed)
- Other targets as needed
