# Goal
Eliminate the repeated stub module pattern across all test files by creating a shared `tests/stubs.nix` module.

## Prerequisites
- None

# Acceptance Criteria
- [ ] Create `tests/stubs.nix` exporting:
  - `hostPlatformStub` — stubs `nixpkgs.hostPlatform` with the current system
  - `moduleArgsStub` — stubs `_module.args` with `pkgs`
  - `darwinEnvStub` — stubs Darwin-specific options (`environment.systemPackages`, `launchd.user.agents`, `system.activationScripts`)
  - `nixosEnvStub` — stubs NixOS-specific options (`environment.systemPackages`, `systemd`, `users`, `networking.firewall`)
  - `optionsStub` — imports `modules/common/options.nix`
- [ ] Refactor `tests/test-services.nix` to use `stubs.nix`
- [ ] Refactor `tests/test-home-manager.nix` to use `stubs.nix`
- [ ] Refactor `tests/test-vllm-mlx.nix` to use `stubs.nix`
- [ ] Refactor `tests/test-microvm.nix` to use `stubs.nix` where applicable
- [ ] Run `devenv tasks run check:lint` and `devenv tasks run test`

# Files

# Notes
Every test file currently repeats:
```nix
stubModules = [
  ../modules/common/options.nix
  { options.nixpkgs.hostPlatform = lib.mkOption { type = lib.types.anything; default = { inherit (pkgs.stdenv.hostPlatform) system; }; }; }
  { config._module.args = { inherit pkgs; }; }
];
```
Darwin tests add more stubs for `environment`, `launchd`, `system.activationScripts`, etc. Centralizing these reduces test boilerplate by ~20 lines per file.
