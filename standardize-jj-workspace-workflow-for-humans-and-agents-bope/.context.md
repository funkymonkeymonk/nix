# Goal
Define and document a consistent jj workspace workflow that both humans and agents follow.
Workspaces should feel like branches (not sibling directories), live in a fixed location,
and be the default way to do isolated development in this repo.

# Direction (decided in design session 2026-05-11)

## Workspace Location
Workspaces live at ~/.jj/workspaces/<repo>/<type>-<topic>-<date>-<id>/ -- never as sibling dirs.
fjj alias and jj-workspace script already implement this -- confirm and document.

## Conceptual Model
- A workspace IS a branch you can cd into
- Multiple changes in flight = multiple workspaces
- Single active change = work directly in main repo on a branch (workspaces optional)
- Agents always use workspaces (isolation requirement)

## Agent Requirements
- Agents MUST use workspaces for all non-trivial changes
- Agent workspace names encode agent identity: feat/agent-openclaw-<topic>
- Agents stop session and clean workspace when done (no leaked workspaces)
- jj-workspace-session TTL handles cleanup automatically
