# Goal
Replace the fragile TAILSCALE_AUTH_KEY environment variable approach with a proper
1Password/opnix integration so Tailscale auth works reliably without manual env var
injection at deploy time.

# Current state
targets/zero/default.nix:90-93 — a systemd oneshot service checks for $TAILSCALE_AUTH_KEY
at activation. If not set, it silently skips Tailscale auth. This is fragile and
undocumented — nothing enforces the env var is present.

# Approach
1. Create a 1Password item for the Tailscale auth key
2. Add opnix secret that writes the key to a file (e.g. /run/secrets/tailscale-auth-key)
3. Update the systemd service to read from the file instead of env var
4. Remove the env var check and silent skip — fail loudly if key is missing

# Acceptance Criteria
- [ ] TAILSCALE_AUTH_KEY env var no longer required at deploy time
- [ ] Tailscale auth key stored in 1Password, managed by opnix
- [ ] systemd service reads from opnix-managed file path
- [ ] Service fails loudly (not silently) if secret is not available
- [ ] devenv tasks run check:all passes
- [ ] zero successfully connects to Tailscale on activation
