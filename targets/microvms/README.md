# OpenClaw + Matrix MicroVM Setup

This directory contains microvm configurations for running a complete AI assistant setup with Matrix integration.

## Overview

- **Matrix MicroVM**: Self-hosted Matrix Synapse homeserver with Element Web client
- **OpenClaw MicroVM**: AI assistant that connects to the Matrix server
- Both use **Opnix** for 1Password secrets management

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                        Host System                               │
│                                                                  │
│  ┌─────────────────────┐      ┌───────────────────────────────┐ │
│  │   Matrix MicroVM    │      │     OpenClaw MicroVM          │ │
│  │   ─────────────     │      │     ────────────────          │ │
│  │   Hostname: matrix  │◄────►│     Hostname: openclaw        │ │
│  │   IP: 10.0.2.15     │      │     IP: 10.0.2.16            │ │
│  │                     │      │                               │ │
│  │   • Synapse         │      │   • OpenClaw Gateway          │ │
│  │   • Element Web     │      │   • Matrix Bot                │ │
│  │   • Port 8008       │      │   • Port 18789                │ │
│  └─────────────────────┘      └───────────────────────────────┘ │
│                                                                  │
│  Secrets from 1Password via Opnix                                │
└─────────────────────────────────────────────────────────────────┘
```

## Prerequisites

### 1. 1Password Service Account Token

Create a 1Password service account and get a token:

```bash
# In 1Password, create a service account and export the token
export OP_SERVICE_ACCOUNT_TOKEN="your-token-here"
```

### 2. Required 1Password Items

Create these items in your 1Password vault. **Each field has detailed notes** explaining:
- What the secret is for
- How to generate/obtain it
- Where it's used
- How to rotate it

See **SECRETS.md** for complete documentation.

#### Matrix Synapse (`op://Homelab/Matrix Synapse/`)

Individual secrets (composed at runtime):
- **signing-key**: Ed25519 signing key for Matrix federation
- **registration-shared-secret**: Shared secret for user registration
- **admin-password**: Password for @admin:matrix.local user
- **openclaw-password**: Password for @openclaw:matrix.local bot

#### OpenClaw (`op://Homelab/OpenClaw/`)

Individual secrets (composed at runtime):
- **zen-api-key**: OpenCode Zen API key (obtain from provider)
- **matrix-access-token**: Matrix bot access token (obtain after Matrix starts)

**Note**: Environment files are generated at runtime from these individual secrets. No combined environment files are stored in 1Password.

## Building and Running

### Build the MicroVMs

On an x86_64-linux machine with Nix:

```bash
# Build Matrix microvm
nix build .#microvm.nixosConfigurations.matrix.config.microvm.declarationRunner --impure

# Build OpenClaw microvm  
nix build .#microvm.nixosConfigurations.openclaw.config.microvm.declarationRunner --impure
```

### Run the MicroVMs

```bash
# Run Matrix microvm
nix run .#microvm.nixosConfigurations.matrix.config.microvm.declarationRunner --impure

# Run OpenClaw microvm (in another terminal)
nix run .#microvm.nixosConfigurations.openclaw.config.microvm.declarationRunner --impure
```

## Configuration Details

### Matrix MicroVM

**Location**: `targets/microvms/matrix.nix`

**Services**:
- Matrix Synapse on port 8008
- Element Web client via nginx on port 80
- SQLite database (for simple single-node setup)

**Secrets Architecture**: Individual secrets are stored in 1Password and composed at runtime:
- `signing-key` → `/var/lib/matrix-synapse/matrix.local.signing.key`
- `registration-shared-secret` → `/var/lib/matrix-synapse/registration_secret`
- `admin-password` + `openclaw-password` → Used directly by user creation script (no env file stored)

**Access**:
- Matrix API: `http://matrix:8008`
- Element Web: `http://matrix`

### OpenClaw MicroVM

**Location**: `targets/microvms/openclaw.nix`

**Services**:
- OpenClaw Gateway on port 18789
- Matrix bot client

**Configuration**:
```json
{
  "agent": {
    "model": "anthropic/claude-sonnet-4-1"
  },
  "channels": {
    "matrix": {
      "enabled": true,
      "homeserver": "http://matrix:8008",
      "userId": "@openclaw:matrix.local"
    }
  }
}
```

**Secrets Architecture**: Individual secrets stored in 1Password, composed at runtime:
- `zen-api-key` → Composed into `/run/openclaw/generated-env` as `ZEN_API_KEY`
- `matrix-access-token` → Composed into `/run/openclaw/generated-env` as `OPENCLAW_MATRIX_ACCESS_TOKEN`
- Environment file is generated by `openclaw-generate-env` systemd service at startup

**Access**:
- Gateway WebSocket: `ws://openclaw:18789`

## Inter-VM Networking

The microvms use user-mode networking by default. For the microvms to communicate:

1. **Host-based networking**: Each microvm gets an IP from the host
2. **DNS resolution**: The host's `/etc/hosts` or local DNS should map:
   - `matrix` → Matrix microvm IP
   - `openclaw` → OpenClaw microvm IP

For production use, consider:
- Setting up a bridge network
- Using static MAC addresses
- Configuring proper DNS

## Usage

### 1. Start Matrix Server

```bash
nix run .#microvm.nixosConfigurations.matrix.config.microvm.declarationRunner --impure
```

Wait for the server to start and create the admin user.

### 2. Create OpenClaw Bot Account

SSH into the Matrix microvm and create an access token:

```bash
ssh root@matrix

# Get an access token for the OpenClaw bot
curl -X POST http://localhost:8008/_matrix/client/r0/login \
  -H "Content-Type: application/json" \
  -d '{
    "type": "m.login.password",
    "user": "openclaw",
    "password": "your-bot-password"
  }'
```

Save the access token to 1Password.

### 3. Start OpenClaw

```bash
nix run .#microvm.nixosConfigurations.openclaw.config.microvm.declarationRunner --impure
```

### 4. Chat with OpenClaw

Open Element Web at `http://matrix` and:
1. Log in with your admin account
2. Create a room
3. Invite `@openclaw:matrix.local`
4. Start chatting!

## Managing Secrets

Opnix automatically syncs secrets from 1Password at boot time.

### Token File

Place your 1Password service account token on the microvm:

```bash
# SSH into the microvm
ssh root@matrix

# Create token file
echo "your-service-account-token" > /etc/opnix-token
chmod 600 /etc/opnix-token

# Restart Opnix to sync secrets
systemctl restart onepassword-secrets
```

### Updating Secrets

When you update secrets in 1Password:

```bash
# On the microvm
sudo systemctl restart onepassword-secrets

# Services will automatically restart to pick up new secrets
```

## Troubleshooting

### Check Service Status

```bash
# Matrix
ssh root@matrix
systemctl status matrix-synapse
journalctl -u matrix-synapse -f

# OpenClaw
ssh root@openclaw
systemctl status openclaw-gateway
journalctl -u openclaw-gateway -f
```

### Check Opnix Logs

```bash
journalctl -u onepassword-secrets -f
```

### Verify Secrets

```bash
ls -la /run/secrets/
cat /run/secrets/matrix-admin-env
```

## Security Considerations

1. **Token Storage**: The 1Password service account token must be manually placed on each microvm
2. **Networking**: Default setup uses user-mode NAT - not suitable for production
3. **Registration**: User registration is disabled by default
4. **Federation**: Federation is enabled but unrestricted - configure whitelist as needed

## Customization

### Adding More Channels

Edit `targets/microvms/openclaw.nix` and add channel configuration:

```nix
services.openclaw.extraConfig.channels = {
  telegram = {
    enabled = true;
    botToken = "{file:/run/secrets/telegram-bot-token}";
  };
};
```

Add the secret to 1Password and Opnix configuration.

### Using PostgreSQL Instead of SQLite

For better performance with many users, switch to PostgreSQL:

```nix
services.postgresql = {
  enable = true;
  ensureDatabases = ["matrix-synapse"];
  ensureUsers = [{
    name = "matrix-synapse";
    ensureDBOwnership = true;
  }];
};

services.matrix-synapse.settings.database = {
  name = "psycopg2";
  args = {
    user = "matrix-synapse";
    database = "matrix-synapse";
    host = "/run/postgresql";
  };
};
```
