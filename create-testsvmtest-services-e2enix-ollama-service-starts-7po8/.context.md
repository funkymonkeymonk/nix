# Goal
Create tests/vm/test-services-e2e.nix that boots a NixOS VM with llm-host role enabled
and verifies the ollama systemd service starts and listens on port 11434.

# Implementation Notes

## Use mkVmTest helper
Requires full ./modules tree to get services/ollama/ module. See parent yak for mkVmTest design.

## The ollama challenge: GPU acceleration
ollama in the llm-host role may require GPU acceleration config. In a QEMU VM:
- Set myConfig.ollama.acceleration = null (or "none") to disable GPU
- ollama should still start and listen even without GPU — it falls back to CPU
- If ollama requires specific hardware even to start, use a stub/mock approach:
  test that the systemd unit is *defined* and *enabled* rather than fully running

## Test cases

### ollama-service-starts
- Enable myConfig.roles.llm-host.enable = true
- Set myConfig.ollama.acceleration to disable GPU
- Boot and assert: systemctl is-active ollama.service OR systemctl is-enabled ollama.service
- Assert: port 11434 is open (machine.wait_for_open_port(11434)) if service can fully start

### ollama-options-applied
- Verify myConfig.ollama.host and port options are reflected in systemd service config
- Check /etc/systemd/system/ollama.service (or equivalent) contains expected values

## Fallback if ollama can't start in QEMU
If full service start is not feasible without GPU:
- Test that the unit file exists: machine.succeed("systemctl cat ollama.service")
- Test that it's enabled: machine.succeed("systemctl is-enabled ollama.service")
- Document the limitation in a comment
